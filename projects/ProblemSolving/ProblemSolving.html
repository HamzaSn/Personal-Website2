Solution

in my first attempt to solve this exciting problem, i literally took the queen and brute-force moves calculation and checking for obstacle in every square in the queen eight directions, for a large input, a very bad case scenario is when there is no obstacle in one or many certain directions, the algorithm will check every square in every direction anyway.

So in order to have a better solution decided to split the problem in two parts : - calculate the edges of the given board relatively to the queen position. - identify the closest obstacles to the queen and calculate the total number of legal moves.

Part One

having the edges of the chessboard for each case before considering the obstacles given a position of the queen and a dimension of the board will be a good step in having an optimal solution. this first function will calculate the steps a queen can make when there is no obstacle and takes dimension of the board n, row position of the queen r_q and column position c_q as input and returns the total steps of the queen, number of steps in each direction and a table containing all the edges , if you want to access only the table or the number of move in a certain direction, you can do that by doing

Part Two

in this part of the solution the first step i took is to define the real obstacles which are the obstacles excluding unnecessary obstacles meaning obstacles that are not in the queens trajectory this step is made easy because we will use the calculated then splitting the result according each direction (right_real_obstacles,left_real_obstacles, ) the next step is identify the closest obstacle by finding the minimum or maximum (according to whether the queen is going up or down, left or right) then finally sum all the differences between the closest obstacles and the queen position for each direction

since we will be iterating through the all the obstacles only once (to find the relevant ones) this second part of the solution has an O(k) time complexity where k is the number of obstacles.
